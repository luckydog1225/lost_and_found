# 一.项目整体核心架构
浏览器 / Postman / 以后的前端
        │
        ▼  HTTP（JSON）
┌───────────────────────────────────────┐
│  app/main.py          应用入口，挂路由   │
│  app/routers/*.py     接口：收请求、返回  │  ← 你写「发启事」主要在这里
│  app/schemas/*.py     接口用的数据形状    │  ← 请求/响应长什么样
│  app/deps.py          公共依赖：DB、当前用户│
│  app/security.py      密码、JWT           │
├───────────────────────────────────────┤
│  app/models/*.py      表在 MySQL 里长啥样 │  ← Post 你已写好
│  app/database.py      连 MySQL、会话工厂  │
└───────────────────────────────────────┘
        │
        ▼
     MySQL（users 表、posts 表）

总结：上面是[网站接口]，下面是[数据库表]，中间用schema翻译JSON，用model翻译表行

# 二.各文件分别是干什么
## 1.入口层——服务从哪启动
app/main.py：创建app，把 auth 等路由 include_router 挂上去；/health 在这里
启动命令本质是：uvicorn app.main:app → 所有请求先进 main.py 里的 app，再分到各个 router

unicorn app.main:app表示
unicorn：ASGI服务器，负责监听端口、接收HTTP请求
app.main：python模块，对应文件app/main.py
:qpp表示该模块里名为app的对象
也就是说：Uvicorn会去app/main.py里找到这个app对象，并把它当成整个Web服务的入口
在这个项目里的对象就是FastAPI这个实例：
app = FastAPI(title = "失物招领")
所以整条命令的意思是：用Uvicorn启动app/main.py里的FastAPI应用

## 2.路由层（routers）——某个URL收到请求后做什么
app/routers：不写表结构，制作业务流程——查重、加密密码、db.add、返回JSON、抛401等

## 3.Schema层——JSON长什么样（给接口用的，不是数据库表）

## 4.依赖与安全——每个请求共用的工具
app/deps.py：get_db()：一次请求一个数据库连接；get_current_user()：从 Header 里 Bearer Token 解析出是谁
app/security.py：hash_password / verify_password；create_access_token / decode_access_token

# 总结一下

办事窗口    真正接单、入库、回执     routers/post.py ← 你问的这个
