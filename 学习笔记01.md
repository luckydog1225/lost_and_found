## 五、遇到的问题与解决方法

> 这部分是笔记核心价值：体现 **问题排查能力**，面试时可作为 STAR 素材。

---

### 问题 1：终端找不到 `mysql` 命令

**现象：**
```
mysql : 无法将"mysql"项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

**原因：**  
MySQL 已安装（服务 `MySQL80` 在运行），但安装后未刷新终端，当前会话读不到更新后的 PATH。

**解决方法：**
1. **重新打开终端**（最简单）
2. 或在当前 PowerShell 刷新 PATH：
   ```powershell
   $env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
   ```
3. 临时使用完整路径：
   ```powershell
   & "C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p
   ```

**收获：** 区分「程序已安装」和「命令行可用」；环境变量修改后需重启终端。

---

### 问题 2：访问 `/health` 返回 404

**现象：** 终端日志显示：
```
GET /health%EF%BC%8C%E5%BA%94%E8%BF%94%E5%9B%9E HTTP/1.1" 404 Not Found
```

**原因：**  
浏览器地址栏粘贴了说明文字，实际访问的是 `/health，应返回`（含中文逗号），而非 `/health`。

**解决方法：**  
地址栏只输入：
```
http://127.0.0.1:8000/health
```

**收获：** HTTP 路由必须精确匹配；404 不一定是代码错，先检查请求 URL。

---

### 问题 3：MySQL 命令拼写错误

**现象 A：** 输入 `SHOW DATABASE` 后出现 `->` 续行提示，命令卡住。  
**原因：** 正确命令是 `SHOW DATABASES;`（有 S，有分号）。

**现象 B：** `USE lost_and_founf` 报错 `Unknown database`。  
**原因：** 数据库名拼写错误，正确为 `lost_and_found`。

**解决方法：**
- SQL 命令注意拼写和结尾分号 `;`
- 卡住时用 `\c` 取消当前输入，或 `EXIT` 重进

**收获：** 养成 `USE 库名;` → `SHOW TABLES;` → `DESCRIBE 表名;` 的固定验证流程。

---

### 问题 4：`init_db` 连接 MySQL 失败（DATABASE_URL 解析错误）

**现象：**
```
Can't connect to MySQL server on 'Zxy16622875265!@localhost'
```

**原因：**  
`.env` 中密码包含特殊字符 `@`，未做 URL 编码。连接串格式为：
```
mysql+pymysql://用户名:密码@主机:端口/数据库名
```
密码里的 `@` 被误解析为「密码结束、主机开始」的分隔符。

**解决方法：**
1. 修正 `.env` 中 `DATABASE_URL` 格式
2. 若密码含 `@`、`#`、`!` 等，需进行 **URL 编码**（如 `@` → `%40`）
3. 真实密码只放 `.env`，`.env.example` 用 `your_password` 占位

**收获：** 连接串是结构化 URL，特殊字符必须编码；敏感信息不进 Git。

## 六、核心知识点速查

### 6.1 三层数据表示

| 层次 | 文件 | 作用 |
|------|------|------|
| 数据库表 | `models/user.py` | 对应 MySQL 中 `users` 表 |
| API 输入/输出 | `schemas/user.py` | 规定 JSON 格式，自动校验 |
| HTTP 接口 | `routers/auth.py` | 处理请求，调用数据库 |

### 6.2 注册接口关键步骤

1. `UserRegister` 校验请求格式
2. `get_db()` 获取数据库会话
3. `db.query(User).filter(...)` 查重
4. `hash_password()` bcrypt 加密
5. `db.add` → `db.commit` → `db.refresh` 写入并读回
6. `UserResponse` 返回（不含密码）

## 九、面试可能问到的问题（提前准备）

1. **FastAPI 和 Flask/Django 有什么区别？你为什么选 FastAPI？**  
   → 可答：自动生成 OpenAPI 文档、原生 async 支持、Pydantic 集成好、性能较好。

2. **为什么密码用 bcrypt 而不是 MD5？**  
   → bcrypt 专为密码设计，带盐、慢哈希，抗暴力破解；MD5 太快且无盐，不安全。

3. **Model 和 Schema 为什么要分开？**  
   → 数据库字段与 API 字段不同（如 password vs password_hash）；返回时不能暴露敏感字段。

4. **`db.add` 和 `db.commit` 的区别？**  
   → add 加入会话待提交队列；commit 才真正执行 SQL 写入数据库。

5. **如果两个用户同时注册相同用户名会怎样？**  
   → 代码层查重 + 数据库 unique 约束双重保障；后者是最后防线。
